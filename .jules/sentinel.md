## 2024-05-18 - Prompt Injection via Unsanitized Git Diff
**Vulnerability:** The AI analysis API took an unsanitized `diff` from the client and injected it into an XML-like tag `<diff>{raw_git_diff_string}</diff>` using `String.prototype.replace('{raw_git_diff_string}', diff)`. This permitted an attacker to inject `</diff>` into the diff body and add arbitrary instructions to the LLM. Furthermore, `diff` could contain regex replacement patterns (e.g., `$&`, `$'`), altering the prompt unintentionally.
**Learning:** Raw user input mapped to XML prompt constructs is a high-severity Prompt Injection risk if closing tags are not sanitized. In Javascript, string interpolation using `.replace()` with user input as the second argument causes bugs or injection if the input contains special string patterns.
**Prevention:** Always escape XML closing tags for user inputs in LLM prompts (e.g. `diff.replace(/<\/diff>/g, '<\\/diff>')`). When using `.replace()` with user-controlled strings, strictly use the callback form (e.g., `.replace(pattern, () => userInput)`) to prevent regex variable injection.
